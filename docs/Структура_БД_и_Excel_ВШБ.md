# Соответствие структуры Excel «ПРОЕКТ - ВШБ.Reg.xlsx» и БД приложения

## 1. Листы Excel

| Лист | Назначение |
|------|------------|
| **БАЗА - Аудиторная** | Основная таблица: дисциплины, ОП, часы по РУП, распределение по преподавателям (лекции/семинары), должность, штат/ГПХ, итоговые часы |
| **БАЗА - Внеаудиторная** | Внеаудиторная нагрузка по департаментам и ППС: вид работы, число работ, норматив, часы |
| **СВОДНАЯ - РУП** | Свод по образовательным программам (ОП) и курсам |
| **СВОДНАЯ - ДЕП** | Свод по департаментам |
| **СВОДНАЯ - ПРЕП** | Свод по департаментам и преподавателям (ППС) |

## 2. БАЗА - Аудиторная: колонки Excel → БД

### 2.1 Дисциплина и программа

| Excel | Таблица/колонка в БД | Примечание |
|-------|----------------------|------------|
| ID | `plan_disciplines.plan_discipline_id` | Уникальный ID строки плана |
| Наименование дисциплины | `plan_disciplines.discipline_name` | |
| Департамент | `departments.name` через `plan_disciplines.implementing_department_id` | |
| Вид дисциплины | `plan_disciplines.discipline_kind` | обязательная/по выбору и т.д. |
| ОП | `educational_programs.name` через `plan_programs` + `plan_discipline_programs` | одна дисциплина может быть в нескольких ОП |
| Уровень (бак, маг, асп) | `educational_programs.education_level` | |
| Курс | `plan_disciplines.course_no` | |
| По выбору (О/В) | можно хранить в `plan_disciplines.discipline_kind` или отдельный флаг | |
| Язык | `plan_disciplines.language` | |
| МКД (да, нет) | `plan_disciplines.mkd` | |
| Зач.ед. | `plan_disciplines.credits` | |
| Модули | `plan_modules` + `plan_disciplines.module_id` | |

### 2.2 Часы по РУП и аудиторные

| Excel | БД | Примечание |
|-------|-----|------------|
| Лекции по РУП | `plan_disciplines.rup_lectures_hours` | |
| Семинары по РУП | `plan_disciplines.rup_seminars_hours` | |
| Потоки | `plan_disciplines.streams_count` | |
| Группы | `plan_disciplines.groups_count` | |
| Лекции | факт: через `assignment_hours` + `work_types` (тип «Лекции») | |
| Семинары | факт: через `assignment_hours` + `work_types` (тип «Семинары») | |
| НИС / ПС / ПОС / СН | `plan_disciplines.aud_nis_ps_sn_hours` или типы в `work_types` | |
| Всего ауд часов | `plan_disciplines.aud_total_hours` | |
| Число студентов | `plan_disciplines.students_count` | |
| Текущий контроль | `plan_disciplines.current_control_hours` | |
| Всего часов | `plan_disciplines.total_hours` | |
| Часы по модулям | `plan_disciplines.hours_module1` … `hours_module4` | |

### 2.3 Преподаватели и распределение

| Excel | БД | Примечание |
|-------|-----|------------|
| ФИО лекции (часы Л) | `teaching_assignments` + `assignment_hours`: преподаватель + роль (лекции) + часы | одна дисциплина — несколько записей (лектор, семинаристы) |
| ФИО семинары (часы С) | то же, тип работы «Семинары» | |
| Ответственный преподаватель | `teaching_assignments.role` = «Ответственный» или отдельное поле | в коде есть роль «Преподаватель» |
| ППС | `faculty_members.full_name` | |
| Должность | в текущей БД: не отдельное поле в `faculty_members` | можно добавить `position` (профессор, доцент и т.д.) |
| Штат / ГПХ | в текущей БД нет | можно добавить `employment_type` в `faculty_members` |
| Часов горловой / аудиторной / текущего контроля / нагрузки | считаются по `assignment_hours` и `work_types` или хранятся в представлении | вид `v_workload_by_worktype` даёт часы по типам работ |

### 2.4 Поля Excel, которых может не быть в БД

- Бюджетная / коммерческая  
- Нераспределенная нагрузка (можно считать: разница между планом и суммой назначений)  
- Доля «отлично» / «хорошо» (качество)  
- СОП, Номер договора, Дата заключения, Сумма выплат, Решение Комиссии по качеству  

При необходимости их можно добавить в `plan_disciplines` или в отдельные таблицы (например, договоры ГПХ).

## 3. БАЗА - Внеаудиторная

| Excel | Рекомендация по БД |
|-------|---------------------|
| Департамент | `departments` |
| ППС | `faculty_members` |
| Внеаудиторная нагрузка (вид) | справочник или текст (например, «КП 1 курс») |
| Бюджетная / Коммерческая | флаг или справочник |
| Число работ | число |
| Норматив, на одну работу | число |
| Число часов | число |

**Вариант:** отдельная таблица `extracurricular_workload` (department_id, faculty_id, work_kind, budget_commercial, work_count, norm_per_work, hours). В текущем коде такой таблицы нет — её нужно ввести при поддержке внеаудиторной нагрузки.

## 4. Сводные листы и работа приложения

- **СВОДНАЯ - РУП** → в приложении: выбор ОП и курса, отчёт по нагрузке по программам/курсам (раздел «Нагрузка», фильтры ОП/курс).
- **СВОДНАЯ - ДЕП** → фильтр по департаментам, отчёт по департаменту (те же данные нагрузки с группировкой по департаменту).
- **СВОДНАЯ - ПРЕП** → раздел «ППС» (uifaculty): список преподавателей по департаменту и их нагрузка (часы по типам работ из `v_workload_by_worktype` и т.п.).

То есть логика «как должно работать» уже совпадает: одна база дисциплин и назначений, а экраны и фильтры дают срезы по ОП, курсу, департаменту, преподавателю.

## 5. Текущие таблицы БД (используемые в коде)

- **app_users**, **user_allowed_ops**, **user_allowed_departments** — пользователи и доступ по ОП/департаментам.
- **study_plans**, **plan_modules** — учебные планы и модули.
- **educational_programs** — ОП.
- **plan_programs** — связь плана и ОП.
- **plan_disciplines** — дисциплины плана (все поля из раздела 2).
- **plan_discipline_programs** — связь дисциплины с программами (одна дисциплина — несколько ОП).
- **departments** — департаменты.
- **faculty_members** — ППС (сейчас: full_name, is_active; при необходимости добавить должность, штат/ГПХ).
- **work_types** — типы работ (лекции, семинары, НИС, текущий контроль и т.д.).
- **teaching_assignments** — назначение преподавателя на дисциплину (plan_discipline_id, faculty_id, role).
- **assignment_hours** — часы по назначению и типу работы (assignment_id, work_type_id, hours).
- Представление **v_workload_by_worktype** — сводная нагрузка (дисциплина, ОП, преподаватель, тип работы, часы и т.д.) для экрана «Нагрузка».

Полная схема создания этих таблиц и представления находится в корне проекта: **schema.sql**. Порядок применения: 1) schema.sql, 2) add_auth_tables.sql, 3) add_aud_total_columns.sql (при необходимости). При старте приложение выполняет CREATE OR REPLACE VIEW v_workload_by_worktype.

## 6. Что сделать, чтобы БД и работа были «как в Excel»

1. **Проверить наличие всех таблиц и представления**  
   Убедиться, что в БД есть перечисленные выше таблицы и представление `v_workload_by_worktype`, и что в нём участвуют ОП, курс, департамент, тип работы, часы.

2. **Добавить при необходимости**  
   - В `plan_disciplines`: бюджетная/коммерческая (если нужно).  
   - В `faculty_members`: должность (position), штат/ГПХ (employment_type).  
   - Таблицу внеаудиторной нагрузки и экран/отчёт под лист «БАЗА - Внеаудиторная».

3. **Импорт из Excel**  
   Реализовать загрузку «БАЗА - Аудиторная» (и при необходимости «БАЗА - Внеаудиторная») в эти таблицы: разбор строк, создание/обновление ОП, департаментов, дисциплин, назначений и часов по типам работ.

Если пришлёте пример заполненного Excel (1–2 строки аудиторной и одна внеаудиторной), можно расписать точный маппинг колонка → поле БД и порядок вставки/обновления.
